# 产品

Paseo 是什么，面向谁，以及发展方向。

## 什么是 Paseo

Paseo 是围绕代理构建的下一代开发环境。一个统一的界面，在桌面、移动端、终端和网页上运行、监控和交互编码代理。

开发工作流正在从手动编辑文件转变为编排代理来完成编辑。Paseo 正是为这种工作流而构建的。

## 核心理念

自由和灵活性。每一项设计决策都源于此：

- **多提供方** —— 使用任何编码代理框架。为每个任务选择合适的模型，随着格局变化自由切换。无供应商锁定。
- **跨设备** —— 桌面、移动端、网页、CLI。在桌前开始工作，在手机上查看进度，从终端编写脚本。
- **自托管** —— 守护进程在你的机器上运行。你的代码、你的密钥、你的环境。无推理加价，无云依赖。
- **尊重用户** —— 无遥测，无强制云服务，无强制账户。
- **开源** —— AGPL-3.0。用户可以检查、分叉和贡献。
- **自带密钥** —— 使用你自己的密钥。利用你的补贴计划和第一方提供方定价。Paseo 在上面不增加任何成本。

## 工作原理

### 项目和工作区

项目在侧边栏中分组，自动从你的文件系统中检测，并在可用时通过 git remote 标记。

每个项目以工作区形式打开。对于 git 项目，默认工作区是主 checkout。用户可以创建额外的工作区，它们是隔离的副本（git worktree），代理在其中工作而不影响主分支。

### 工作区内部

工作区是一个灵活的画布：

- 在分栏中并排启动多个代理
- 在代理旁边打开终端
- 在同一工作区内混合搭配提供方

### 守护进程

Paseo 是一个客户端-服务器系统。守护进程（Node.js）在你的机器上运行，管理代理进程，并通过 WebSocket 实时流式传输输出。客户端连接到守护进程——本地或远程。

这种架构意味着：

- 守护进程可以在任何机器上运行：笔记本电脑、虚拟机、远程服务器
- 多个客户端可以同时连接
- 关闭应用后代理继续运行

## 目标用户

任何构建软件的人：

- 关心拥有自己的工具和数据
- 使用多个 AI 提供方并希望自由切换
- 在真实项目上运行代理处理真实任务
- 希望从多个设备工作

## 随着时间复利增长的东西

- **信任** —— 每日出现，公开交付，保持开源。赢得缓慢，失去迅速。
- **社区贡献** —— 代码、打包、技能、代理配置。贡献者成为倡导者。
- **生态系统** —— 技能、集成、共享配置。社区构建的内容让平台更有价值。

## 战略赌注

1. **模型商品化。** 价值转移到编排层。最好的模型每月都在变——工作流层保持不变。
2. **多提供方胜出。** 没有单一提供方能一直领先。开发者想要每个任务的最佳模型。
3. **守护进程即基础设施。** 服务器/客户端架构支持在任何地方部署。
4. **开源的寿命超越融资。** 开源社区是韧性的。贡献者成为倡导者。

## 当前状态（2026 年 5 月）

- 桌面（Electron）、移动端（iOS/Android）、网页、CLI
- 内置提供方：Claude Code（Agent SDK）、Codex（app-server）、GitHub Copilot（ACP）、OpenCode、Pi、OMP
- 一键 ACP 提供方目录：CodeWhale、Cursor、Hermes、Qwen Coder、Kimi Code 等——以及自定义 ACP 提供方
- 语音模式：免提口述提示或讨论问题
- MCP 服务器将守护进程暴露给其他代理（create_agent、send_agent_prompt、调度、终端、工作树、工作区重命名）
- 定时代理（cron 风格触发器）通过应用、CLI 和 MCP
- 高频发布（每周多次）
- 社区贡献涵盖打包、提供方和 bug 修复
- 关键用户体验：分栏、快捷键自定义、工作区模型、应用内浏览器
