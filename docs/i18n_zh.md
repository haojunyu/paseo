# 国际化

Paseo 客户端 UI 翻译位于 `packages/app/src/i18n` 中。

## 支持的语言

- `en`
- `ar`
- `es`
- `fr`
- `ja`
- `pt-BR`
- `ru`
- `zh-CN`

持久化的应用语言设置为 `"system" | "ar" | "en" | "es" | "fr" | "ja" | "pt-BR" | "ru" | "zh-CN"`。当系统语言映射到支持的语言时，`"system"` 跟随设备或浏览器语言；不支持的系统语言回退到英语。日语从系统语言 `ja` 和日语区域语言映射。巴西葡萄牙语从系统语言 `pt-BR` 和裸 `pt` 映射；其他葡萄牙语区域语言在显式添加之前保持不支持。

## 翻译范围

翻译客户端拥有的 UI 文案：标签、按钮、空状态、确认文本以及本地的状态/错误包装。

不要翻译代理输出、守护进程输出、终端内容、文件路径、提供方名称、模型名称、命令名称、用户撰写的文本、代码块、日志或原始协议/服务器错误文本。

## 添加文案

英文源字符串位于 `packages/app/src/i18n/resources/en.ts`。简体中文字符串位于 `packages/app/src/i18n/resources/zh-CN.ts`。

对于已迁移的屏幕和组件，使用 `useTranslation()` 并将翻译后的文本传递给 UI 原语。低级原语（如 `<Button>`）除非自己拥有所渲染的文本，否则不导入翻译状态。

将资源键按产品界面分组，而非按组件机制分组。

运行：

```bash
npx vitest run packages/app/src/i18n/resources.test.ts --bail=1
```

一致性测试会捕获英文与每种支持语言资源之间缺失的键。

## 迁移顺序

客户端 UI 翻译是分阶段进行的，这样每次迭代可以迁移完整的本地文案集群，并保持审查聚焦。

1. 应用外壳和共享 UI 镶边：通用操作、标题、弹出面板、命令中心和客户端拥有的 toast 状态。
2. 编辑器和代理工作流：编辑器输入、代理控制、权限提示、计划审批和代理面板包装状态。
3. 设置扩展：外观、快捷键、集成、权限、诊断、关于、项目设置、主机设置和提供方诊断。
4. 工作区和面板：设置/文件/浏览器/终端包装文案、文件浏览器本地状态、导入会话流程和剩余的本地 toast/错误包装文本。

在已迁移的界面中，当相邻标签由客户端拥有时，不要留下混合语言的标签。将整个本地文案集群一起迁移。
